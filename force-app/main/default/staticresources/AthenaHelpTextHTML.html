Search Tips: 


You can improve your search results by adding operators to the search term. Here are a few ways to effectively use operators with your search terms:

 

Query syntax example (Simple Queries)

Returns all items

term1 OR term2

containing either term1 or term2

term1 NOT term2

containing term1 but not term2

term1 -term2

containing term1 but not term2

"term1 term2 term3"

containing exactly the phrase between double-quotes (Use double quotes for error messages or any term with special characters like [],<>,::)

+term

containing exactly term, not other words sharing the same root

term*

containing term or any word starting with term and ending with any characters [Eg, SIP*]

 

Query syntax example  (Field Queries)

Returns all items

@title=term

with the title field containing term [@title=How To]

@title==term

wit h exactly matching term

@date=yesterday

with the date field being yesterday's date

@date=2015/01/01..2015/03/31

with the date field for a range of dates

 

Advanced (Field Queries)

Returns all items

@athenaProduct=product name

 

 

with the product field containing the product name
[@athenaProduct=PowerCenter]

@athenaProductVersion=product version

with the product version field containing the product version
[@athenaProductVersion=PowerCenter 9.0.1]
@athenaAuthor=authorname with the articles written by the authorname mentioned
@athenaauthor= (authorname1, authorname2, authorname3) ​with either of the enumerated values
[ @athenaauthor= (Tom, Tim, Bob) ]
@infaproblemdescription= *term* with term mentioned anywhere in the Problem Description section of an article [@infaproblemdescription= *exception* ]
​​@athenaauthor=authorname @date=2016/04/01..2016/06/30

 with articles written by the author during the specfied time range. 
[@athenaauthor=Ram @date=2016/04/01..2016/06/30 ]
